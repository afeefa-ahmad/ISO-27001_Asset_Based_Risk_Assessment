# Project Introduction

## What this project is

This repository documents an Information Security Risk Assessment carried out
against the ISO/IEC 27001:2022 standard. It works through a complete assessment
from start to finish for a small, fictional online retailer, and presents the
result the way a working GRC analyst would: a set of clear, connected documents
that move from understanding the business, to modelling its systems, to
identifying and treating its risks.

It is a portfolio project. The company is invented, but the method, the
structure, and the documents follow how a real ISO 27001 risk assessment is
actually performed.

## Why it exists

This project is built to show a single, coherent assessment where every part 
follows from the one before it. The goal is to demonstrate not just that individual 
artifacts can be produced, but that the reasoning behind a risk assessment is understood 
as a whole.

## What it demonstrates

- Scoping an ISMS and understanding the context of an organization
- Modelling a company's systems and data flows in an architecture diagram
- Building an asset inventory from that architecture
- Identifying threats and vulnerabilities for each asset
- Analyzing and scoring risk in a consistent, repeatable way
- Treating risk and mapping it to ISO 27001 Annex A controls
- Producing a Statement of Applicability
- Reporting the results clearly to decision makers

## The standard it follows

The assessment is aligned to ISO/IEC 27001:2022, the international standard for an
Information Security Management System (ISMS). It uses an asset based method, the classic
approach described in ISO/IEC 27005, where risks are identified by first
understanding the assets that need protecting. The project is scoped to ISO 27001
only. Other standards like PCI DSS that could apply to an online retailer are deliberately
left out so that the work stays focused on a single standard.
